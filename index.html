Django

Lista de comandos para utilizar Django:

  1. Visita el sitio oficial haciendo click aqui
  2. Antes que nada tenemos que tener Python instalado. Abrir la Terminal de nuestro sistema o de VSC.
  3. Instalando Django:
      Con Windows, podemos instalarlo bajo el comando pip install Django​
      Para comprobar si se instaló correctamente, escribimos python
      Luego import django, y por último django.VERSION 
  4. Para crear un proyecto:
      Crear una carpeta nueva. Nombrarla PythonProyecto1​, por ejemplo.
      Iniciar VSC o tu editor ( o incluso cmd) y pararte en esa carpeta. 
      Escribir django-admin startproject Proyecto1
      Donde Proyecto1 será una nueva carpeta que se creará en PythonProyecto1.
      Con este último comando se creará una carpeta y un archivo manage.py
      Nos paramos en nuestro proyecto, Proyecto1, cd Proyecto1
      Tipeamos python manage.py migrate
      Verificamos con: ​ python manage.py runserver para iniciar el servidor.
  5. Para crear una aplicación:
      Crear proyecto:django-admin startproject ProyectoCoder
      Entramos al proyecto con cd ProyectoCoder 
      Creamos la app: python manage.py startapp AppCoder
  6. Para crear un modelo:
      Vamos a models.py que se encuentra en nuestra aplicación.
      Vamos a settings,​ INSTALLED_APPS y agregamos nuestra APP.​
      Verificamos si se creo la base de datos correctamente: python manage.py check AppCoder​
  7. Para crear la estructura de la base de datos:
      python manage.py sqlmigrate AppCoder 0001
      python manage.py migrate
  8. Instalando DB Browser:
      Visita el sitio oficial haciendo click aqui
  9. Agregar información a mi base de datos:
      En la Terminal escribimos python manage.py shell
      Luego vamos a importar el modelo, en este caso la clase Curso: from AppCoder.models import Curso​
      Creamos un Curso:​ curso =Curso(nombre="Python", camada=23800)​
      Guardamos el Curso en nuestra base de datos: curso.save()
  10. Al hacer un cambio en el modelo SIEMPRE debemos hacer las migraciones:
      python manage.py makemigrations
      python manage.py migrate​
      Suele ser útil cuando uno genera cambios muy grandes en el modelo borrar la base de datos y volverla a crear.
      En el caso que en algún momento deban hacer eso, es necesario recordar que tendrán que generar otro superusuario.​
      Debiendo crear un username, password y email.
      python manage.py createsuperuser​
  11. Control de versiones:
      pip install -r requirements.txt
